CSRF Testing

What is CSRF (Cross-Site Request Forgery) Testing?

CSRF Testing is the process of checking whether a web application properly validates user actions to prevent unauthorized commands being performed on behalf of an authenticated user.


Why is it Important in Ethical Hacking and Penetration Testing?

A vulnerable site may allow attackers to trick users into making unwanted changes (like updating a password or email) without their consent. This can lead to:

  • Account takeover
  • Data manipulation
  • Unauthorized actions performed in the background
  • Session abuse through social engineering

Proper CSRF protection ensures that every critical action requires intentional user verification.

Python Code: CSRF Testing


import requests

# Target endpoint that performs a state-changing action (e.g., change password, update email)
target_url = "http://example.com/change-email" # Replace with actual target
cookies = {'sessionid': 'your-session-id'} # Replace with valid session cookie if needed

# Simulated CSRF request without token
data = {
'email': 'attacker@example.com' # Payload you want to test
}

print("[+] Sending simulated CSRF request...")

response = requests.post(target_url, data=data, cookies=cookies)

# Check response for signs of success or protection
if "email updated" in response.text.lower():
print("[!] Potential CSRF vulnerability detected!")
else:
print("[-] CSRF protection might be in place or action blocked.")
