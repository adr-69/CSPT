Web Application Scanning

What is Web Application Scanning?

Web Application Scanning is a technique used to find exposed endpoints, hidden files, and possible vulnerabilities on a website. It involves probing common URLs and checking server responses to identify weak spots or misconfigurations.


Why is it Important in Ethical Hacking and Penetration Testing?

Web apps often expose login pages, admin panels, or configuration files. Ethical hackers and penetration testers use scanners to:

  • Discover hidden or forgotten pages
  • Identify security headers and server info
  • Spot outdated technologies or frameworks
  • Detect misconfigured or exposed resources

This helps developers and security teams fix weaknesses before malicious hackers can exploit them.

Python Code: Web Application Scanner


import requests

# Target website  #this my sample website
target_url = "https://adr-69.github.io/-/PWS.htm"  # Replace with the website you want to scan

# Common paths to scan (add more if needed)
common_paths = [
    "/", "/admin", "/login", "/dashboard", "/wp-login.php", "/config", "/.git", "/robots.txt"
]

print(f"[+] Scanning {target_url} for common endpoints...\n")

for path in common_paths:
    url = target_url + path
    try:
        response = requests.get(url, timeout=5)
        code = response.status_code
        print(f"{url} => [{code}]")

        # Show server header if found
        server = response.headers.get('Server')
        if server:
            print(f"    Server: {server}")
        x_powered = response.headers.get('X-Powered-By')
        if x_powered:
            print(f"    X-Powered-By: {x_powered}")

    except requests.exceptions.RequestException:
        print(f"{url} => [Error connecting]")

print("\n[+] Scan complete.")
